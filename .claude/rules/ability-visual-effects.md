# Візуальні ефекти здібностей (усі шляхи)

Будь-яка здібність, що має візуальний ефект, має відчуватись видовищно — не як
одинокий партикл, а як поєднання кількох партиклів/форм/звуку. Правило діє на
всі 22 шляхи, не лише Sun.

## Обов'язково

- **Кілька шарів, не один партикл**: ефект здібності комбінує щонайменше два
  елементи з різних категорій нижче (напр. промінь + кільце, або партикл-шлейф +
  звук + кільце при імпакті). Голий одинарний `spawnParticle`/`playSound` без
  пари — тільки для дрібних допоміжних акцентів (іскра при ударі, тік пасивки),
  ніколи як єдиний ефект активної здібності.
- **Форма, не хмара**: використовуй складові примітиви з `IVisualEffectsContext`
  (`playBeamEffect`, `playCircleEffect`, `playSphereEffect`, `playHelixEffect`,
  `playWaveEffect`, `playExplosionRingEffect`, `playAlertHalo`) замість
  хаотичного розкиду партиклів без форми.
- **Колір за брендингом шляху**: колір партиклів (`Particle.DUST` +
  `DustOptions`) бере відтінок з `PathwayBranding.liquidOf(pathwayName)` —
  не хардкодь колір власного шляху; див. `.claude/rules/pathway-branding.md`.
- **Тематичний звук**: звук відповідає темі шляху/здібності (церемоніальний,
  темний, механічний тощо), не голий ванільний бойовий звук (arrow/hit) як
  головний.
- **Просторова логістика**: ефекти "згори" йдуть від точки високо над ціллю до
  цілі; ауру/німб піднімай над головою (`add(0, 2.0, 0)`), а не хмарою навколо
  всього тіла.

## Заборонено

- ❌ Один голий партикл (`Particle.END_ROD`/`CLOUD`/будь-який) як єдиний ефект
  здібності — читається як "нічого не сталось".
- ❌ Чисто разовий `spawnParticle` без форми (лінія/коло/промінь/сфера) для
  здібностей з великим імпактом.
- ❌ Хардкодити колір шляху напряму в ability-класі — лише через
  `PathwayBranding`.

## Виняток

Навмисно скромні здібності найслабших Sequence (напр. Sun Sequence 9, Bard) —
там менша видовищність відповідає лору, не порушення правила.
