# Повідомлення комітів

- Формат — Conventional Commits: `type(scope): subject`, наприклад `feat(creatures): ambient spawner near converged players`.
- **Тільки англійською** — і заголовок, і body.
- `subject`: короткий (≤ 72 символів разом із префіксом), наказовий спосіб (`add`, `fix`, `replace` — не `added`/`adds`), без крапки в кінці, **без тире** як роздільника чи замість двокрапки.
- Заголовок чітко називає **конкретну** зміну чи нову механіку. Заборонено розмите: `update code`, `improvements`, `misc fixes`.
- Один коміт = одна логічна зміна. Якщо в subject проситься "and" між двома незв'язаними речами — розбий на два коміти.
- Типи: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`, `perf` + проєктний `balance` (зміни ігрового балансу). `scope` — підсистема: `creatures`, `behaviors`, `spells`, `rules`, назва pathway тощо (як у git log).
- Body — опційний, лише коли треба пояснити **чому** (мотивація, компроміс, наслідок для сумісності), а не переказати diff.
