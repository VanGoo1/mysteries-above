# Робота з Git

- **НІКОЛИ не створюй git worktree.** Для ізольованої роботи над фічею роби звичайну
  окрему гілку (`git checkout -b feat/<опис>`) у тому самому робочому дереві, а не
  `git worktree add`. Не запускай `EnterWorktree`, `git worktree add` чи режими
  ізоляції `worktree` для сабагентів. Якщо інструмент чи скіл пропонує worktree —
  відхили й зроби гілку.
- Нові гілки фіч називай за конвенцією репозиторію: `feat/<kebab-опис>`
  (як `feat/ritual-magic-divination`), а не `worktree-*`.
- Комітити чи пушити — лише коли користувач попросив. Формат повідомлень — див.
  `commit-messages.md`.

## Гілку створюй ПЕРШИМ кроком, ще до спеки

**Уся робота однієї теми живе в ОДНІЙ гілці — включно з її документами.** Спека
(`docs/superpowers/specs/*.md`), план (`docs/superpowers/plans/*.md`), код, тести,
правила `.claude/rules/*.md` і оновлення `CLAUDE.md` — це одна тема, отже одна гілка.

Порядок такий, і саме в такому порядку:

1. `git checkout main && git pull` — свіжа база.
2. `git checkout -b feat/<kebab-опис>` — **гілка створюється ДО того, як написано
   перший рядок спеки**, а не після того, як почався код.
3. Спека → план → код → тести → правила — усе комітиться в цю гілку.

**Ніколи не комітить спеку чи план у `main`** — навіть якщо це «поки що просто
документ» і код почнеться завтра. `main` рухається лише мерджем гілки.

**Чому це важливо (реальний випадок, 21.07.2026):** три docs-коміти
(`ritual-magic-divination` спека, план, shapeshifting-доповнення) написали прямо в
`main`, а розробку потім вели в окремій гілці, куди ті самі патчі приїхали
черіпіком з іншими SHA. У підсумку `main` тижнями висів «попереду origin на 3
коміти» з роботою, яка вже була в гілці, і виглядало це як загублені коміти.
Довелось звіряти патчі через `git cherry -v` і скидати `main`.

Ознака, що правило порушено: `git status` на `main` каже «ahead of origin/main by
N commits», хоча жодної завершеної фічі туди не мерджили. Перевір
`git cherry -v <гілка-фічі> main` — якщо коміти позначені `-`, вони вже є в гілці,
і локальний `main` треба скинути (`git reset --hard origin/main`), а не пушити.
