# Економіка Характеристик — Спек 6c: Таємні організації (дизайн)

**Дата:** 2026-07-16
**Статус:** затверджено до реалізації
**Парасольковий роадмап:** [2026-06-26-ingredient-economy-roadmap.md](2026-06-26-ingredient-economy-roadmap.md)
**Попередник:** [спек 6b — церкви](2026-07-12-ingredient-economy-churches-design.md)

## Огляд і межі

Таємні організації (ордени) — тіньовий конкурент ортодоксальних церков. 25 орденів уже
закладені в `InstitutionRegistry` (6b) як дані; цей спек дає їм механіку. Чотири стовпи
суперництва:

1. **Рейди на сховища церков** — інфільтрація храму, крадіжка з `ChurchVault`;
2. **Шпигунство** — подвійні агенти (церква + орден одночасно) зливають розвіддані й
   саботують варіння; ціна провалу — вигнання з церкви;
3. **Замахи на священиків** — храм лишається без послуг на години;
4. **Конкуренція за прогресію** — орден дає те, чого церква не дає ніколи: сирі матеріали
   (включно з Характеристиками), заборонені знання, унікальні послуги. Але не готові зілля —
   «вари сам».

**Поза межами:** механіка Якоря для Seq 2 (рішення брейнштормінгу: поки не робимо);
активне «полювання на шпигунів» гравцем-детективом (нема чесної механіки виявлення —
викриття системне); ініціація гравця без шляху через орден (монополія дуелі церков 6b);
фізичні лігва орденів (присутність — предмет-талісман).

## Ухвалені рішення (з брейнштормінгу)

| Питання | Рішення |
|---|---|
| Стовпи суперництва | Усі чотири: рейди, шпигунство, замахи, конкуренція за прогресію |
| Вступ | Два канали: генеричний предмет-ключ «Шифроване послання» (вибір ордену під свій шлях) + персистентні запрошення за агресивні/вперті вчинки від тематичного ордену |
| Присутність | Предмет «Талісман зв'язку»: ПКМ будь-де → меню ордену; лігв немає |
| Ростер | Усі 25 орденів живі; доступність гравцю — за `Institution.acceptsPathway` |
| Ранги | Ранг = послідовність гравця (5 щаблів за Seq), НЕ вклад; очок немає взагалі |
| Нагороди | Need-based прохання до куратора після завдання (фавор): рецепт якщо не знаєш наступного; кілька інгредієнтів (не всі) або Характеристика — залежно від ваги завдання; легкі завдання можуть не дати нічого |
| Економіка нагород | Гібрид: матерія — зі схованки ордену (наповнюється рейдами/доставками + малий сід), знання — напряму від куратора (прецедент дуель-ініціації 6b) |
| Рейд | Фізична інфільтрація реального храму вночі: канал-таймер, шанс тривоги → варта; здобич — реальні предмети зі сховища церкви |
| Біль для церкви | «Болюче, але відновне»: вкрадене зникає назавжди, але рейд обмежений лімітом здобичі й кулдауном храму; священик відроджується за N годин |
| Шпигунство | Злив маніфеста сховища (RECON) + саботаж варіння (SABOTAGE); викриття = вигнання з церкви (незворотне, через `leave()`) |
| Контррозвідка | Постійне доручення «Захисти храм»: тривога рейду сповіщає членів церкви, вбивство рейдера = великий вклад |

## Вступ і членство

**Канал 1 — «Шифроване послання»** (custom item):

- Джерела: лут данжів/рідкісних скринь (`global_loot.yml`), підпільний ринок, рідкісний
  дроп з міфічних істот. Ресурспак-текстура як в інших custom-предметів.
- ПКМ → меню вибору серед орденів, що приймають шлях гравця
  (`Institution.acceptsPathway`; порожні доступи = приймає будь-який шлях). Вступ через
  `ConfirmationMenu` з попередженням про однобічність (як у церков). Послання
  споживається при вступі.

**Канал 2 — запрошення за вчинки:**

- Тригери прив'язані тематично до орденів (мапа тригер→орден — код, поруч із реєстром):
  убив апекс-істоту → орден, що покриває шлях її групи; серія PvP-вбивств Потойбічних →
  криваві секти; зупинив рампейджера → тощо. Пороги — `orders.invites.*` у config.
- Запрошення **персистується** (`order-memberships.json`): гравець офлайн чи пропустив
  чат — запрошення чекає; нагадування при вході на сервер; прийняти можна будь-коли
  (клікабельне повідомлення або `/order invites`). Не згорає.
- Запрошення від ордену, що не приймає шлях гравця, не генерується (тест).

**Членство (`OrderMembership`):**

- Один орден + одна церква паралельно (слот закладений у 6b). Гравець **без шляху**
  вступити не може — ранг виводиться з послідовності; ініціація новачків лишається
  монополією церков.
- **Ранг = послідовність**: Seq 9–8 «Пішак», 7–6 «Вістря», 5–4 «Довірений», 3–2
  «Магістр», 1–0 «Прихований Владика». Ранг гейтить вагу завдань і стелю прохань.
  Просунувся послідовністю → ранг росте сам (нуль стану).
- **Талісман зв'язку** — custom item, видається при вступі; ПКМ будь-де → меню ордену.
  Загубив → `/order talisman` перевидає безкоштовно з кулдауном. Меню без талісмана
  недоступне — «фізичність» ордену.
- **Вихід — необоротний** (дзеркало церков): `abandonedOrders` назавжди; кулдаун на
  вступ в інший орден (`orders.rejoin-cooldown-days`). `/order leave` — двокрокова
  (`/order leave confirm`).

## Завдання і фавори (куратор)

**Куратор** — не фізичний NPC, а наративна персона меню талісмана (ім'я з лор-пулу
ордену; репліки — action bar/чат, стиль Посередника ринку). «Людина, що поручається
за тебе».

**Генерація** — чистий `OrderTaskGenerator` + той самий `TaskQuota` (вікно
`orders.tasks.refresh-hours`, набір `orders.tasks.max-active`, квота
`orders.tasks.sets-per-window`; правила збереження прогресу при скиданні — як у церков).
Кожне завдання має **вагу**: `LIGHT` / `STANDARD` / `MAJOR`.

| Тип | Опис | Вага | Кому |
|---|---|---|---|
| DELIVER | принести інгредієнти у схованку (здача через меню) | LIGHT–STANDARD | усім |
| HUNT | вбити істот домену ворожої церкви (фолбек «будь-які» як у 6b) | LIGHT–STANDARD | усім |
| RAID | інфільтрація храму (див. нижче) | MAJOR | усім; лише коли в світі є сайти церков |
| ASSASSINATE | замах на священика | MAJOR | ранг «Довірений»+ |
| RECON | злити маніфест сховища своєї церкви | STANDARD | лише подвійним агентам |
| SABOTAGE | зірвати варіння замовлення своєї церкви | MAJOR | лише подвійним агентам |

LIGHT-завдання інколи (config-шанс) **не дають нічого** — куратор лише «кивне»: орден
перевіряє терплячість. Це заявлена атмосфера, не баг.

**Фавор** — виконане завдання створює персистентний запис із вагою завдання. У меню
куратора гравець витрачає фавор на прохання; доступні опції рахує чисте правило
**`FavorOptions`** від ситуації гравця (головний юніт-тест спеку):

- **Не знає рецепту своєї наступної послідовності** → може просити **рецепт**
  (знання напряму через `RecipeUnlockService`, повз схованку — прецедент
  дуель-ініціації 6b). Фавор STANDARD+.
- **Знає рецепт** → **до 2 інгредієнтів** цього рецепта зі схованки (STANDARD; свідомо
  не всі основні), або **Характеристику**[свій шлях, наступна посл.] зі схованки
  (лише MAJOR + ранг «Довірений»+).
- **Інформація** (будь-який фавор): підказка де полювати апексів свого шляху (біоми/
  структури з `CreatureNamer`); якщо орден має свіжі розвіддані — маніфест сховища
  церкви (що красти).
- **Послуги** (MAJOR): зняття кулдауну вступу до церкви/ордену; «чисті документи» —
  разова легенда, з якою церква приймає гравця, чий шлях поза її доменом. Чорних
  списків (`abandonedChurches`/`abandonedOrders`) НЕ обходить ніщо — необоротність 6b
  священна.

Нема в схованці — куратор перелічує брак («принеси здобич — буде з чого платити»),
фавор **не згорає**. Фавори стакаються, персистуються.

## Операції в храмах

**Рейд (RAID):**

- Завдання називає конкретну церкву з розміщеним сайтом. Виконати може лише носій
  активного RAID-завдання.
- Виконання: прийти до храму **вночі**, клік талісманом по блоку в радіусі сайту →
  **канал-таймер** (`orders.raid.channel-seconds`; прогрес у action bar, треба лишатися
  в зоні). Щосекунди — шанс **тривоги** (`orders.raid.alarm-chance`): спавн істот-варти
  домену церкви (MythicMobs, гейтвей як у дуелі), священик здіймає крик. Тривога не
  зриває злом — варта просто вбиває; смерть або втеча з зони = провал, кулдаун храму.
- **Успіх** → зі сховища церкви (реальне списання через `ChurchService`) в інвентар
  падає **здобич**: ліміт позицій за вагами (`orders.raid.loot.*`: інгредієнти часті,
  книги рідше, Характеристика — джекпот). Свіжі розвіддані ордену → маніфест видно
  перед рейдом, здобич цінніша, шанс тривоги нижчий.
- Здобич — реальні предмети: завдання зараховане лише після **здачі у схованку** через
  меню. Лишив собі — завдання провалене, фавору нема, предмети твої (свідома чесна
  спокуса). Сира Характеристика в руках рейдера — легальний канал: це «украдена в 6c»
  з інваріанта 6b.
- Кулдаун операцій на конкретний храм (`orders.raid.temple-cooldown-hours`, дефолт 24,
  спільний для рейду й замаху) + ліміт здобичі = «болюче, але відновне». Поповнення
  сховища — лише пожертви/доставки членів церкви (як у 6b).

**Замах (ASSASSINATE):**

- Лише за активним завданням, ранг «Довірений»+. Удар виконавця по NPC-священику →
  священик «тікає» (despawn), спавниться **бойовий охоронець храму** (істота домену
  церкви, висока послідовність). Вбив охоронця → замах зараховано; вбили тебе → провал,
  священик повертається одразу.
- Наслідок: храм без послуг на `orders.assassination.priest-respawn-hours` (дефолт 12,
  реальний час): NPC відсутній, меню недоступне, видача замовлень чекає (варіння у фоні
  триває). Потім священик повертається (наявний механізм респавну з `church-sites.json`).

## Подвійні агенти і контррозвідка

Шпигунські завдання генеруються **лише** члену церкви + ордену одночасно. Обидва
виконуються фізично через священика **своєї** церкви:

- **RECON:** взаємодія зі священиком за активним RECON → знімок маніфеста сховища
  пишеться у стан ордену як **свіжі розвіддані** з TTL (`orders.recon.ttl-hours`).
  Спільні для всього ордену — підсилюють чужі рейди цієї церкви.
- **SABOTAGE:** короткий канал біля священика → одне активне замовлення іншого члена
  цієї церкви **затримується** на `orders.sabotage.delay-hours` (не знищується — без
  прямого грифу). Замовник бачить «у варильні щось пішло не так».

**Викриття:** шанс на кожну шпигунську дію (`orders.exposure.recon-chance` <
`orders.exposure.sabotage-chance`) і на провалений рейд **своєї** церкви. Викриття →
церква виганяє через наявний механізм `leave()`: вклад/ранг/замовлення згорають,
церква назавжди в `abandonedChurches` (необоротність 6b — кара); членам церкви онлайн —
broadcast «Викрито зрадника: <нік>». Членство в ордені не страждає.

**Контррозвідка** — нове постійне доручення церков «**Захисти храм**»: тривога рейду
сповіщає всіх онлайн-членів цієї церкви (координати). Вбив рейдера до кінця злому →
великий вклад (патерн «зупинити рампейджера»), рейд провалено.

## Схованка ордену і економіка

**`OrderStash`** — дзеркало `ChurchVault` (itemKey → кількість), але **невидима**
членам: вміст знає лише куратор (перелічує брак при проханні).

- **Входи:** здобич рейдів (здача через меню), DELIVER-завдання, **малий одноразовий
  сід** (`orders.stash.seed.*`) — розширення задокументованого винятку емісії 6b;
  позначка в роадмапі.
- **Виходи:** лише прохання-фавори (інгредієнти, Характеристики).
- **Інваріант збереження (юніт-тест):** кожна одиниця схованки — у схованці або видана
  фавором. Сира Характеристика потрапляє гравцю **лише** як здобич рейду (тимчасово, до
  здачі) або як MAJOR-фавор зі схованки. Інваріант сховища церкви (6b) розширюється:
  списано у варіння АБО **вкрадено рейдом**.
- Знання (рецепти, інформація) — повз схованку, напряму від куратора: знання — не
  матерія, прецедент — дуель-ініціація 6b.

## Wiring, конфіг, команди

- **`domain.organizations`** (чистий, уже в PURE_DOMAIN): `OrderMembership`,
  `OrderRank` (Seq→щабель), `OrderTask` (+типи/вага), `OrderTaskGenerator`,
  `Favor`, `FavorOptions`, `OrderStash`, `RaidOutcome` (математика здобичі/тривоги),
  `Invitation` + тригер-правила.
- **`application`:** `SecretOrderService` — оркестратор (instance-поля, гідрація з
  репозиторіїв, persist після КОЖНОЇ мутації — патерн `ChurchService`); `RaidSession` —
  живий злом (канал, тривога, варта — патерн `DuelSession`, власний таск). Точки дотику
  з `ChurchService`: списання сховища при здобичі, затримка замовлення, викриття через
  `leave()`, зникнення/повернення священика (через `ChurchPriestService`).
- **`infrastructure`:** `OrderConfig` (`orders.*`, всі дефолти в коді);
  `order-memberships.json` (членства, фавори, запрошення, `abandonedOrders`, кулдауни
  храмів) і `orders-state.json` (схованки + розвіддані з TTL) — каркас
  `GatheringSnapshotRepository`; `OrderMenu` (triumph-gui); предмети «Шифроване
  послання» і «Талісман зв'язку» — custom-items + ресурспак.
- **`presentation`:** `OrderListener` (ПКМ талісманом/посланням, злом, замах,
  RECON/SABOTAGE, kill-прогрес HUNT, тригери запрошень); команда `/order`
  (`invites`, `talisman`, `leave`/`leave confirm`, `info`) — гравецька.
- **`config.yml` `orders.*`:** `invites.*` (пороги тригерів), `tasks.refresh-hours` /
  `max-active` / `sets-per-window`, `raid.channel-seconds` / `alarm-chance` / `loot.*` /
  `temple-cooldown-hours`, `assassination.priest-respawn-hours`,
  `sabotage.delay-hours`, `recon.ttl-hours`, `exposure.*`, `stash.seed.*`,
  `favor.*` (гейти ваг/рангів), `rejoin-cooldown-days`, `talisman.reissue-cooldown-minutes`.

## Тестування

- **Юніт (чистий домен):** `FavorOptions` — усі ситуаційні гілки (без рецепта → рецепт;
  з рецептом → інгредієнти ≤2 або Характеристика за MAJOR; гейти рангу; LIGHT без
  нагороди); `OrderTaskGenerator` — шпигунські лише агентам, RAID лише за наявності
  сайтів, ваги масштабуються, HUNT-фолбек; `OrderRank` — мапінг Seq→щабель;
  `RaidOutcome` — ліміт здобичі, вплив розвідданих на тривогу/цінність;
  **інваріант схованки** (сід + внески = у схованці + видано фаворами); запрошення —
  лише від орденів, що приймають шлях; необоротність виходу.
- **ArchUnit:** без змін (`domain.organizations` уже в PURE_DOMAIN) — не завозити Bukkit.
- **In-server чекліст:** послання → вступ (підтвердження) → запрошення за вчинок
  (персистується, нагадується) → завдання → рейд уночі (канал, тривога, варта, захист
  храму членом церкви) → здача здобичі → фавор (рецепт/інгредієнти/Характеристика/брак)
  → подвійний агент: RECON → маніфест підсилив чужий рейд → SABOTAGE → викриття
  (вигнання з церкви, broadcast) → замах (священик зник, храм без послуг, повернувся) →
  `/order leave confirm` (необоротність) → втрата талісмана → перевидача → рестарт
  сервера (все відновлене).

## Перевикористання (не переписуємо)

- `InstitutionRegistry` (6b) — ростер орденів готовий; `Institution.acceptsPathway`.
- `TaskQuota`, патерн `ChurchTaskGenerator` — вікно/набори/квота завдань.
- `ChurchService.leave()` + `abandonedChurches` — механізм вигнання при викритті;
  патерн необоротності для `abandonedOrders`.
- `ChurchVault`/`ChurchStateRepository` — модель схованки; `ChurchService` — списання
  сховища при здобичі.
- `ChurchPriestService` — despawn/respawn священика при замаху.
- `RecipeUnlockService` — видача рецепта-фавора (прецедент `completeTrialInitiation`).
- `MythicCreatureGateway` — варта рейду, охоронець замаху, kill-прогрес HUNT.
- `CreatureNamer` — інформація «де полювати».
- `ConfirmationMenu`, triumph-gui, `MarketItemNamer` — UI.
- `GatheringSnapshotRepository` — каркас JSON-персистентності.
- Custom-items + ресурспак — послання/талісман.
- `DuelSession`/`DuelArenaProvider`-патерни — `RaidSession` (сесія з власним таском).
