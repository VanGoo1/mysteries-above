# Економіка Характеристик та інгредієнтів — роадмап

**Дата:** 2026-06-26
**Статус:** парасолькова візія (декомпозиція на спеки). Кожен спек нижче проектується й
реалізується окремо (свій спек → план → реалізація).

## Об'єднавчий принцип

У LOTM сила скінченна: характеристик у світі обмежена кількість, і вони **циркулюють**, а не
з'являються з нічого. Серце плагіна — єдина **«Економіка Характеристик»**: кожне джерело —
це канал постачання в один спільний обіг, а не ізольована фіча.

Три ролі предметів:
- **Характеристика** (головний дефіцитний компонент) — кристалічна есенція, ключована
  **(шлях × послідовність)**. Незнищенна, рідкісна.
- **Допоміжні інгредієнти** — трави/кров/реагенти; поширені.
- **Знання** (рецепт) — уже реалізовано через `RecipeUnlockService` + книги рецептів; це і є
  «Карти/Сланці Богохульства». Без знання рецепта зварити не можна.

## Модель рецепта (фундамент)

```
РЕЦЕПТ(шлях, Seq) = [основні інгредієнти] + [допоміжні інгредієнти]
```

Зварити можна **двома шляхами** (Характеристика — *опціональна заміна*, не обов'язковий гейт):

1. **Класичний:** усі оригінальні основні + усі допоміжні.
2. **Через Характеристику:** 1× `Характеристика[шлях, Seq]` замість *усіх* основних + ті самі
   допоміжні.

Кожен інгредієнт у наборі тегується як **основний** або **допоміжний**. Матчер приймає набір,
якщо присутні всі допоміжні **і** (всі основні **АБО** одна Характеристика потрібного шляху+рівня).

## Канали постачання (розділені за типом і рідкістю)

| Канал | Що дає | Статус |
|---|---|---|
| Природа (форедж) — трава/листя/дерева | Допоміжні інгредієнти (поширені) | новий |
| Звичайні скрині | Базові інгредієнти, нижчі Seq | є |
| Данжі / археологія / спец-структури | Рідкісніші інгредієнти, рецепти, зрідка зілля | є (розділити за рідкістю) |
| Полювання — звичайні міфічні істоти | Переважно інгредієнти (осн./доп.) | новий (A) |
| Полювання — найсильніші (апекс) істоти | **Характеристики** | новий (A) |
| Смерть Beyonder (PvP/PvE) | **Характеристика** його шляху+Seq | новий (B) |

**Інваріант розподілу:** Характеристики з'являються **лише** з найсильніших джерел
(апекс-істоти, смерть Beyonder). Звичайні інгредієнти — звідусіль, але **розділено за рідкістю
джерела**: трава → найдешевше; звичайні скрині → базове; данжі/структури → рідкісне.
Характеристики прибираємо зі звичайного лут-фону.

## Граф каналів

```
РЕЦЕПТ = 1× Характеристика[шлях, Seq] (опц.)  +  N× Допоміжні
              │                                       │
   ┌──────────┴──────────┐                            │
A. Апекс-істоти     B. Смерть Beyonder                │
   (рідкісний дроп)     (PvP/PvE)                      │
   └─── дефіцитне ядро ──┘                             │
                                                       │
Природа/скрині/данжі ──► Допоміжні + рецепти + зрідка зілля
C — компас до найближчого джерела Характеристики
D/E — перерозподіл (ринок/церкви); F — Сефіроти (ендґейм)
```

## Послідовність спеків

1. **Спек 1 — Фундамент.** Тип `Характеристика[шлях, Seq]`; теги основний/допоміжний на
   інгредієнтах; логіка опціональної заміни у зварюванні; прибрати Характеристики зі
   звичайного лут-фону. *Передумова для A і B.*
2. **Спек 2 — B (вилучення).** Смерть Beyonder (гравець/NPC) → конденсація його
   Характеристики його шляху+послідовності. Найменший споживач фундаменту; валідовує модель.
3. **Спек 3 — A (полювання).** Кастомні міфічні істоти: звичайні → інгредієнти; апекс →
   Характеристики.
4. **Спек 4 — Природа + розділення лута.** Форедж допоміжних з трави/листя/дерев; тиризація
   лут-джерел понад звичайні скрині.
5. **Спек 5 — C (Закон Конвергенції).** Компас/тяжіння до найближчого джерела потрібної
   Характеристики (істота / випала характеристика / Beyonder того ж чи сусіднього шляху).
6. **Спек 6 — D/E (перерозподіл).** Збори Потойбічних / підпільний ринок (анонімний обмін) та
   церкви/таємні організації (членство → видача знань і матеріалів за службу).
   - 6a (ринок) — **реалізовано**.
   - 6b (церкви) — **реалізовано**: реєстр 10 церков + 25 орденів, автоспавн храмів біля
     сіл, членство з вкладом/рангами, служба (полювання/доставка/рампейджери/пожертви),
     замовлення зілля зі сховища церкви, ініціація гравця без шляху. Свідомий виняток
     емісії: сховища церков засіваються готовими рецептами/інгредієнтами/Характеристиками
     (`seedVaultIfAbsent`) — це керований інжект матеріалів у замкнене коло служби, не
     фонова емісія (гравцям сирі Характеристики зі сховища не видаються, лише списуються у
     варіння замовлень). Правило: `.claude/rules/church-organizations.md`.
7. **Спек 7 — F (Сефіроти).** Ендґейм-резервуари сили (вище Seq 0).

## Перевикористання (не переписуємо)

- **Рецепт-система** (`RecipeUnlockService` + книги) = Карти/Сланці Богохульства.
- **`global_loot.yml` + лут-лістенери** — фоновий канал; у Спеку 1/4 розділяємо за рідкістю.
- **`IItemResolver` / `CustomItemRegistry`** — усі нові дропи через наявну фабрику предметів.
- **`PathwayManager` + `Sequence`** — B і C спираються на «шлях+рівень» сутності.

## Поза межами роадмапу (поки що)

- Точний баланс ваг/шансів кожного каналу — фіналізується в межах відповідного спека.
- Сефіроти (F) — лише на найвищих рівнях, проектується останнім.
