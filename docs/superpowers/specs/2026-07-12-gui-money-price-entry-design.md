# GUI-пікери суми та кількості для ринку

**Дата:** 2026-07-12
**Гілка:** feat/currency-coin-items

## Проблема

Виставлення ціни/кількості на підпільному ринку (`MarketMenu`) відбувається через
чат-промпти (`ChatPromptService`). Формат ціни — «`<фунти> <коппети>`»: перше число
ЗАВЖДИ трактується як фунти. Гравець, що хоче 15 **коппетів**, природно пише «15» і
отримує 15 **фунтів** (= 300 коппетів) — тихий, у 20 разів дорожчий баг. Загалом
чат-ввід посеред GUI-меню незручний і неочевидний.

## Мета

Перевести **весь** ввід чисел на ринку в GUI: клікабельні кнопки, живий підсумок,
кнопка «Скасувати» в самому вікні. Жодного чат-вводу.

## Обсяг: чотири точки чат-вводу в `MarketMenu`

1. **Ціна монетами** (`promptMoney`, р.334) — лот / оферта / зустрічна ціна (усі через
   `chooseConsideration`).
2. **Доплата монетами (буст)** у бартері (`withBoot`, р.364) — нуль дозволений.
3. **Кількість штук** при створенні замовлення (`openKnownIngredients`, р.216) — 1–64.
4. **Кількість штук** у бартерній вимозі (`openBarterPicker`, р.357) — 1–64.

## Дизайн

### Компоненти (нові, `infrastructure.ui` — шар ефектів)

**`MoneyPicker`** — чест-GUI (3 ряди) з кнопками +/- , повертає `PoundMoney`.
- Внутрішній стан сесії — один `int` (усього коппетів). Кнопки додають/віднімають:
  `+10ф`=+200, `+1ф`=+20, `+5к`=+5, `+1к`=+1; мінус симетрично з клампом на 0.
  Перевищення 19к само «перекочується» у фунти, бо `PoundMoney` тримає суму в
  коппетах, а `format()` («2 ф 15 к») малює це автоматично.
- Центр — `GOLD_NUGGET` із живим підсумком `price.format()`; поряд `✔ Підтвердити`
  та `✖ Скасувати`.
- Прапорець `allowZero`: ціна → `false` (Підтвердити відмовляє й підсвічує «потрібна
  сума > 0», поки 0); буст → `true`.
- API: `open(Player, String title, boolean allowZero, Consumer<PoundMoney> onConfirm,
  Runnable onCancel)`.

**`AmountPicker`** — той самий патерн для цілого в `[min,max]` (кнопки `+10/+1/-10/-1`,
кламп у межі), повертає `int`. Для кількості штук — `[1,64]`, старт на `min`.
- API: `open(Player, String title, int min, int max, IntConsumer onConfirm,
  Runnable onCancel)`.

**Рендер:** кожен пікер тримає власний `Gui` та стан у приватному inner-класі
`Session`; клік кнопки мутує стан і оновлює слот підсумку через
`gui.updateItem(row,col,...)` + `gui.update()` (патерн уже вживаний у `Spellcasting` /
`RecipeBookMenu` — без close/open-мерехтіння). Стиль кнопок/звуків — як у
`ConfirmationMenu`.

### Перепровід у `MarketMenu`

- `promptMoney(...)` → `MoneyPicker.open(allowZero=false)`; `onConfirm` →
  `action.accept(Consideration.money(price))`; `onCancel` → назад у
  `chooseConsideration`. Покриває лот/оферту/зустрічну автоматично.
- Бартерний буст → `MoneyPicker.open(allowZero=true)`; `onCancel` → закрити (скасувати
  дію).
- Обидві кількості → `AmountPicker.open(1,64)`; `onConfirm` → відповідний виклик
  `gatheringService`; `onCancel` → назад у попередній пікер/меню.
- Прибрати мертві після цього хелпери: `withPrice`, `withBoot`, `parseAmount`,
  `PRICE_HINT`, поле `prompts` та його конструкторний параметр.

### Прибирання `ChatPromptService`

Після конверсії `ChatPromptService` (клас-Listener) і його `parsePrice` не мають
жодного користувача. Прибрати повністю як мертвий код:
- видалити клас `presentation.listeners.ChatPromptService` і його тест
  `ChatPromptServiceTest`;
- зняти поле/створення/геттер у `ServiceContainer`;
- зняти реєстрацію лістенера в `MysteriesAbovePlugin.registerEvents()`;
- оновити застарілий коментар у `GatheringListener` (посилався на порядок пріоритетів
  із `ChatPromptService`).

## Архітектура / чистота

Пікери — суто Bukkit-ефект → `infrastructure.ui`; балансних формул не додають, тож
`domain` не чіпаємо. Грошовий VO — наявний `PoundMoney` (тримає суму в коппетах,
`format()`/`of` уже є). Арифметика клампа тривіальна й лишається в пікері — це не
«правило балансу», юніт-тест не потрібен. Перевірка — in-server, як і решта GUI-ефектів.
