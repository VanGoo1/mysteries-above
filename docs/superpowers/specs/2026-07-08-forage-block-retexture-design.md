# Форедж 2.0 — «зачаровані» блоки замість нод-сутностей (дизайн)

**Дата:** 2026-07-08
**Статус:** затверджено до реалізації
**Парасольковий роадмап:** [2026-06-26-ingredient-economy-roadmap.md](2026-06-26-ingredient-economy-roadmap.md)
**Замінює механізм ноди зі** [Спеку 4 — Природа + тиризація](2026-06-29-ingredient-economy-nature-loot-design.md)
(тиризація луту з того спека **не чіпається**; переробляється лише канал фореджу).

## Огляд

Поточний форедж спавнить **видиму 3D-модель інгредієнта** (ItemDisplay + Interaction, збір ПКМ)
«всередині» рослинності. Переробляємо на **«зачаровані» блоки**:

- спавнер, як і зараз, періодично шукає вегетацію біля гравця, але замість сутностей **фізично
  підміняє блок на блок-донор**, який ресурспак перемальовує в «зачаровану» версію (сяйна трава,
  сяйне листя);
- над зачарованим блоком тихо іскряться **ледь помітні частинки** (`GLOW`, 1–2 шт. кожні ~2 с);
- гравець **ламає** блок → ванільний дроп скасовується, падає **лише допоміжний інгредієнт**
  (+ звук/партикл збору);
- не зібрали за TTL → блок **тихо відновлюється** в оригінал.

**Механізм текстур — донор-блоки.** Ресурспак не може додати новий блок, лише перемалювати наявні,
тому в світ ставиться блок, якого в Оверворлді природно не буває (нижчесвітні/ендові рослини), і його
текстура перемальовується **цілком** (тільки PNG, без JSON-моделей):

| Оригінал | Донор (дефолт) | Файл текстури |
|---|---|---|
| Трава / папороть | `WARPED_ROOTS` | `assets/minecraft/textures/block/warped_roots.png` |
| Квіти | `CRIMSON_ROOTS` | `crimson_roots.png` |
| Кущ / інше | `NETHER_SPROUTS` | `nether_sprouts.png` |
| Листя (будь-яке) | `AZALEA_LEAVES` | `azalea_leaves.png` |

Свідома жертва: донори виглядатимуть «зачаровано» й у рідному вимірі (Незер/Люш-печери).
Не кожна квітка має власну текстуру — 3–4 **архетипи** вигляду; мапінг конфігурований.

**Нове коло цілей:** до наземної флори додається **листя крон** (окремий список у конфігу).
Земля/повні непрозорі блоки — поза межами цього спека.

### Інваріанти

- Форедж далі видає **тільки допоміжні** інгредієнти з `forage.yml` (біом-тематично, всім гравцям).
- Дроп при ламанні зачарованого блока — **лише інгредієнт**: ванільний дроп (насіння/ягоди/саджанці)
  скасовується (`setDropItems(false)`).
- Живі ноди існують **лише в завантажених чанках**; будь-який нештатний кінець (вивантаження,
  disable, креш) закінчується відновленням оригінального блока.
- Неігрове знищення ноди (поршень, вибух, пожежа, потік води) → відновлення/зняття **без** дропу
  інгредієнта — щоб не фармили автоматикою.

## Архітектура (rules vs effects)

Чисте ядро не змінюється: `ForageSelector` (`domain.forage`) і біомні таблиці лишаються як є.
Нова чиста логіка одна — **мапінг донорів** (оригінал → донор з дефолтами й overrides): маленький
VO `ForageDonorMap` у `domain.forage` (рядкові імена матеріалів, без Bukkit) з юніт-тестом.
Уся мутація світу — infrastructure/presentation, перевіряється in-server.

### `ForageNode` (infrastructure.forage) — тепер block-based

Замість пари сутностей — запис про підмінений блок:

- поля: позиція (`Location`/блок), **оригінальний `BlockData`**, id інгредієнта, `createdAtMillis`;
- `place()` — підміна блока на донора (листя-донор — з `persistent=true`, щоб не осипалось);
- `restore()` — повернення оригінального `BlockData`;
- двоблокова флора (`TALL_GRASS`, `LARGE_FERN`): нижня половина → донор, верхня → повітря;
  `restore()` повертає **обидві** половини.

### `ForageNodeCodec` — перепрофілювання на PDC чанка

Ентіті-теги більше не потрібні. Codec серіалізує записи нод у **PDC чанка** (позиція + рядковий
`BlockData.getAsString()` оригіналу): додає запис при `place()`, знімає при `restore()`/зборі,
читає «осиротілі» записи при завантаженні чанка (креш-відновлення, див. нижче).

### `ForageNodeLocation` — + пошук листя

Наявний пошук наземної флори (на рівні ніг) доповнюється сканом крон: випадкові зміщення
`dx,dz`, згори (`getHighestBlockAt`) вниз кілька блоків, кандидат — блок із списку `leaves`.

### `ForageNodeSpawner` (infrastructure.schedulers) — той самий каркас

- `start()`/`stop()`, головний тік з інтервалом `interval-seconds` — без змін по структурі;
- анти-накопичення рахується **по реєстру позицій** (без сканування сутностей);
- **другий легкий тік** кожні `particle-period-ticks` (дефолт 40): для кожної живої ноди
  1–2 частинки `Particle.GLOW` над блоком;
- TTL-прюн → `restore()` замість `remove()`;
- `stop()` відновлює всі живі ноди (викликається з `stopSchedulers()` в `onDisable`).

### `ForageHarvestListener` (presentation.listeners) — переїзд на блокові події

- `BlockBreakEvent`: позиція в реєстрі спавнера → `setDropItems(false)`, дроп інгредієнта через
  `customItemService`, звук + партикл збору, зняття ноди з реєстру і з PDC чанка (блок ламається
  в повітря штатно — `restore()` не потрібен);
- краї — відновлення/зняття **без дропу**: `BlockPistonExtendEvent`/`RetractEvent`,
  `EntityExplodeEvent`/`BlockExplodeEvent`, `BlockBurnEvent`, `BlockFromToEvent` (вода/лава),
  `LeavesDecayEvent` (страхування попри `persistent=true`);
- `ChunkUnloadEvent` → ноди цього чанка відновлюються й знімаються достроково;
- `ChunkLoadEvent` (+ прохід по вже завантажених чанках в `onEnable`): «осиротілі» записи в PDC
  чанка (лишились після крешу) → відновити блок, почистити PDC.

### Що видаляється

- Спавн `ItemDisplay`/`Interaction` у `ForageNode`, ПКМ-збір через `PlayerInteractEntityEvent`;
- ентіті-PDC частина `ForageNodeCodec`;
- 3D-моделі інгредієнтів у ресурспаку **лишаються** — вони й далі використовуються для самих
  предметів (дроп, інвентар).

## Дані

### `forage.yml` (доповнення)

```yaml
forage:
  spawn:
    # ... наявні ключі без змін ...
    leaves:                    # нові цілі — листя крон
      - OAK_LEAVES
      - BIRCH_LEAVES
      - SPRUCE_LEAVES
    particle-period-ticks: 40  # тік «ледь помітних» частинок над нодами
  donors:                      # оригінал -> донор (ресурспак малює донора «зачарованим»)
    default-plant: WARPED_ROOTS
    default-leaves: AZALEA_LEAVES
    overrides:                 # опційний точковий мапінг
      POPPY: CRIMSON_ROOTS
      SWEET_BERRY_BUSH: NETHER_SPROUTS
```

Валідація матеріалів (цілі, донори) — у `ForageConfigLoader`: warning + пропуск невалідних.
Біомні таблиці `biomes:` — без змін.

## Дуал-пак `mysteries.zip` (датапак + ресурспак в одному архіві)

Датапак **фізично не може** передати клієнту текстури; але один zip може бути водночас
датапаком (сервер читає `data/`) і ресурспаком (клієнт читає `assets/`).

- **Джерело правди в репо** — наявна тека `resourcepack/`: додаються текстури донорів
  (`assets/minecraft/textures/block/{warped_roots,crimson_roots,nether_sprouts,azalea_leaves}.png`).
  У репо кладуться **помітні заглушки**; користувач замінює на свої PNG. JSON-моделі не потрібні —
  перемальовується текстура донора цілком.
- **Збірка** — скрипт `scripts/build-pack.ps1`: бере `mysteries.zip` (його `data/` — джерело правди
  для структур, лишається як є) і вливає в нього весь `resourcepack/` (`assets/` + вже наявні
  3D-моделі інгредієнтів) → **один пак** на роздачу. `pack.mcmeta` виставляється з діапазоном
  форматів, що задовольняє і сервер (data-формат, зараз `min_format: 88`), і клієнт
  (resource-формат); точні числа звіряються з фактичною версією сервера/клієнта при реалізації.
- **Роздача** — оновлений zip хоститься за прямим URL; `server.properties`:
  `resource-pack=`, `resource-pack-sha1=` (README ресурспаку вже описує процес). У
  `world/datapacks/` кладеться той самий файл.

## Wiring (`ServiceContainer` + `MysteriesAbovePlugin`)

- `ForageConfigLoader` читає нові секції (`leaves`, `particle-period-ticks`, `donors`) →
  `ForageConfig` розширюється; `ForageDonorMap` конструюється з конфіга;
- конструктор `ForageNodeSpawner` — ті самі залежності (codec тепер chunk-PDC);
- `ForageHarvestListener` отримує спавнер (доступ до реєстру) + codec; реєструються додаткові
  події (блокові, чанкові) — той самий лістенер-клас;
- порядок `onDisable` не змінюється: `stopSchedulers()` → відновлення всіх нод.

## Тестування

- **Юніт (чистий domain, без Bukkit):**
  - `ForageSelectorTest` — без змін, лишається зеленим;
  - **`ForageDonorMapTest`** — дефолт для рослини/листя; override перемагає дефолт; невідомий
    оригінал → дефолт відповідного класу.
- **In-server (ручне):**
  1. Ходити біомами → трава/квіти/листя біля гравця перетворюються на донорів із кастомною
     текстурою; видно рідкі GLOW-частинки.
  2. Зламати зачарований блок → падає **лише** інгредієнт (без насіння/саджанців); звук/партикл.
  3. Не чіпати ноду TTL → блок тихо повертається в оригінал (перевірити й двоблокову траву).
  4. Поршень/вибух/вода по ноді → блок відновлено/знято, інгредієнт **не** випав.
  5. `stop`/disable → усі ноди відновлені; вивантаження чанка (відійти далеко) → відновлено.
  6. Симульований креш (kill процесу) → після рестарту блоки лікуються при завантаженні чанків
     (PDC чиститься).
  7. Зібраний `mysteries.zip`: клієнт приймає ресурспак, структури датапаку далі генеруються.

## Перевикористання (не переписуємо)

- `ForageSelector`, біомні таблиці, `ForageConfigLoader`-каркас, `ForageNodeSpawner`-каркас
  (`start`/`stop`, per-player try/catch) — лишаються;
- `CustomItemService` — резолв інгредієнта в `ItemStack` для дропу;
- PDC-патерни (`CreatureCodec` та ін.) — взірець для chunk-PDC серіалізації;
- тиризація луту (`LootTier`, `filterByTier`, лістенери) — не чіпається;
- `resourcepack/` + README — база дуал-паку.
