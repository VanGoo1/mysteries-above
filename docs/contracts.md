# Магічне засвідчення (Sun, Посл. 6)

Нотаріус Сонця укладає магічно засвідчені угоди. Здібність `Contract`
(`pathways.sun.abilities`) — тонкий вхід: дивлячись на гравця, відкриває **Золотий
Сувій** (`ContractMenu`) і повертає `deferred()`. Уся UI/ефект-логіка — в
`ContractMenu`; ресурси списуються не при відкритті, а в момент реального ефекту.

## Чотири дії Золотого Сувою

- **Мир** (`ContractTerm.PEACE`) — сторони не можуть шкодити одна одній. Персиститься.
- **Клятва** (`ContractTerm.DEBT`, у UI «Клятва») — ціль (боржник, `partyB`) обіцяє віддати
  предмет кастеру (кредитору, `partyA`) до строку. Зобов'язання = предмет у руці кастера
  (`amount`+`item`+`deadlineEpochMillis` у params). **Кастер обирає предмет із затримкою**:
  після кліку по плитці має вікно `CONSENT_SECONDS`, щоб перекласти потрібний предмет у руку
  й підтвердити присіданням (`promptHeldItem`) — інакше схопився б диск-тригер здібності.
  **Виконання**: боржник присідає й ПКМ по кредитору, тримаючи обумовлену кількість
  (`ContractListener.onPlayerInteractEntity` → `findActiveDebt` → перенос предмета +
  `settle()` без кари). Збіг предмета — за матеріалом (як зберігала Клятва). Строк вийшов
  без виконання → `breach()`. Персиститься.
- **Обмін** — безпечний обмін предметом. Кастер обирає предмет тим самим вікном
  `promptHeldItem` (перекласти в руку + присісти), потім ціль підтверджує. **Без застави,
  validate-at-commit**: пропозиція
  зберігає лише знімок стака в пам'яті (`ContractMenu.pendingOffers`), а перевірка й
  переміщення — одна синхронна транзакція в момент згоди цілі. Жоден предмет не покидає
  інвентар до цього кроку → дублювання й втрата структурно неможливі. **Не персиститься**,
  окремої сесії/таску немає.
- **Засвідчення вміння** — підтримка союзника: множник шкоди через
  `context.amplification().amplifyDamage(allyId, …)` (діє на здібності союзника-Sun) +
  тимчасова підписка `subscribeToTemporaryEvent` на наступний мілі-удар союзника
  (+шкода, +святий бонус, підпал) — працює для союзника будь-якого шляху. **Не
  персиститься.**

Якщо між кастером і ціллю вже є активний контракт — сувій додає плитку **Розірвати**
(`settle()` за згодою).

## Церемонія підпису

Мир/Клятва/Обмін завершуються ритуалом `runSigningCeremony`: обидві сторони коротко
знерухомлюються (`SLOWNESS`+заборона стрибка ~1 с), повертаються обличчям одна до одної,
з'являються золоті письмена (`playScriptureAura`) і Сонячний стовп, що спускається
(`playDescendingSunPillar`), лунає небесний дзвін (`BLOCK_BELL_*`). **Контракт стає
`ACTIVE` лише по завершенню церемонії** (`onComplete` → `sign()`).

## Доменний бекенд (незмінний)

- **Домен** — `domain.contracts` (`PURE_DOMAIN`, нуль Bukkit): `ContractTerm`
  (`PEACE`/`DEBT`) і `Contract` — мутабельний стейт-машина `PROPOSED→ACTIVE→
  SETTLED/BREACHED`, гейти переходів + валідація params. Юніт — `ContractTest`.
- **Контекст** — `IContractContext` / `context.contracts()` (обгортка над
  `ContractService`): `propose`/`cancelProposal`/`sign`/`settle`/`findActiveBetween`.
- **Сервіс** — `ContractService`: `Map<UUID, Contract>` + `JSONContractRepository`
  (`contracts.json`). Персистяться лише `ACTIVE`/`SETTLED`/`BREACHED`; `PROPOSED` живе
  тільки в пам'яті.

## Порушення → Божественна кара

`ContractListener` виявляє порушення (удар між сторонами `PEACE`; спливання строку
`DEBT` **без виконання** — боржник встиг би віддати предмет через sneak+ПКМ) і кличе
`ContractService.breach()`, що виконує `DivinePunishment.punish()`:

1. свята блискавка (`playHolyLightning` — лише візуал, без вогню/шкоди);
2. true damage повз броню (пряме `setHealth(health − X)`);
3. печатка **всіх** здібностей на 10–15 хв (`AbilityLockManager.lockPlayer`);
4. **Зламаний Сонячний Диск** над головою на час печатки (`playBrokenSunDisc`).

`DivinePunishment` (`application.services`) — **перевикористовуваний ранер покарання**
для будь-яких майбутніх механік, не лише контрактів. Кари — упорядковані приватні кроки:
додати новий штраф = метод + рядок виклику. Прогрес `Beyonder`'а кара не чіпає.

## Як додати новий термін контракту

1. Значення в `ContractTerm` + гілка params у `Contract.validateParams`.
2. Плитка + потік у `ContractMenu` (`beginX`), підпис `ContractMenu.termLabel`.
3. Порушення — гілка в `ContractListener` (подієва як `PEACE`, або періодична як `DEBT`).

## Заборони

- ❌ Bukkit/`net.kyori` у `domain.contracts` (ArchUnit `PURE_DOMAIN`).
- ❌ Застава/custody в обміні — лише validate-at-commit (дублювання = баг).
- ❌ Хардкод покарання в `ContractService` — лише через `DivinePunishment`.
- ❌ Персистити `PROPOSED`-контракти чи стан обміну/засвідчення.
